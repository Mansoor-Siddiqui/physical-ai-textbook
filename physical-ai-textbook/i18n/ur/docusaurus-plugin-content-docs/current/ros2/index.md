## ROS2 کیا ہے؟

**ROS2 (روبوٹ آپریٹنگ سسٹم 2)** روبوٹ ایپلیکیشنز بنانے کے لیے ایک اوپن سورس مڈل ویئر فریم ورک ہے۔ اس کے نام کے باوجود، ROS2 آپریٹنگ سسٹم نہیں ہے—یہ سافٹ ویئر لائبریریز اور ٹولز کا مجموعہ ہے جو آپ کو روبوٹ سافٹ ویئر بنانے میں مدد کرتا ہے۔

### ROS2 کیوں؟

| خصوصیت | فائدہ |
|--------|-------|
| **معیاری بنانا** | سینسرز، ایکچویٹرز، اور الگورتھمز کے لیے مشترکہ انٹرفیسز |
| **ماڈیولریٹی** | پیچیدہ سسٹمز کو قابل انتظام، دوبارہ قابل استعمال اجزاء میں تقسیم کریں |
| **کمیونٹی** | پیکجز کا وسیع ایکو سسٹم اور فعال کمیونٹی سپورٹ |
| **ریئل ٹائم سپورٹ** | ڈیٹرمنسٹک، ریئل ٹائم ایپلیکیشنز کے لیے ڈیزائن کیا گیا |
| **ملٹی پلیٹ فارم** | Linux، Windows، macOS، اور ایمبیڈڈ سسٹمز پر چلتا ہے |
| **DDS فاؤنڈیشن** | مضبوط کمیونیکیشن کے لیے ڈیٹا ڈسٹریبیوشن سروس پر بنایا گیا |

### ROS2 بمقابلہ ROS1

ROS2 نے ROS1 کی حدود کو دور کیا:

```
ROS1 کی حدود              →    ROS2 حل
─────────────────────────────────────────────────────
ناکامی کا واحد نقطہ        →    تقسیم شدہ آرکیٹیکچر
صرف Linux                →    کراس پلیٹ فارم سپورٹ
ریئل ٹائم سپورٹ نہیں      →    ریئل ٹائم قابل
سیکیورٹی خدشات           →    بلٹ ان سیکیورٹی (SROS2)
Python 2 انحصار          →    جدید Python 3
```

## بنیادی تصورات

### نوڈز

**نوڈ** ROS2 میں کمپیوٹیشن کی بنیادی اکائی ہے۔ ہر نوڈ ایک واحد، ماڈیولر مقصد کے لیے ذمہ دار ہے۔

```python
# minimal_node.py - Python میں ایک سادہ ROS2 نوڈ
import rclpy
from rclpy.node import Node

class MinimalNode(Node):
    def __init__(self):
        super().__init__('minimal_node')
        self.get_logger().info('ROS2 سے سلام!')
        
        # ایک ٹائمر بنائیں جو ہر سیکنڈ فائر ہو
        self.timer = self.create_timer(1.0, self.timer_callback)
        self.counter = 0
    
    def timer_callback(self):
        self.counter += 1
        self.get_logger().info(f'ٹائمر فائر ہوا: {self.counter}')

def main(args=None):
    rclpy.init(args=args)
    node = MinimalNode()
    
    try:
        rclpy.spin(node)  # نوڈ کو چلتا رکھیں
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### ٹاپکس

**ٹاپکس** نوڈز کے لیے غیر مطابقت پذیر طور پر پیغامات کا تبادلہ کرنے کی نامزد بسیں ہیں۔ پبلشرز پیغامات بھیجتے ہیں؛ سبسکرائبرز انہیں وصول کرتے ہیں۔

```
┌──────────────┐                    ┌──────────────┐
│   پبلشر     │                    │  سبسکرائبر  │
│    نوڈ      │                    │    نوڈ      │
│              │     /camera/image  │              │
│   publish()──┼───────────────────▶┼──callback()  │
│              │                    │              │
└──────────────┘                    └──────────────┘
```

### سروسز

**سروسز** نوڈز کے درمیان مطابقت پذیر، ریکویسٹ-ریسپانس کمیونیکیشن فراہم کرتی ہیں۔

```
┌──────────────┐                    ┌──────────────┐
│   کلائنٹ    │    /add_numbers    │    سرور     │
│    نوڈ      │                    │    نوڈ      │
│              │   ریکویسٹ (a, b)  │              │
│   call() ────┼───────────────────▶│              │
│              │   ریسپانس (sum)   │              │
│   ◀──────────┼────────────────────┼── callback() │
└──────────────┘                    └──────────────┘
```

### ایکشنز

**ایکشنز** طویل عرصے سے چلنے والے کاموں کے لیے ہیں جنہیں فیڈبیک کی ضرورت ہوتی ہے اور منسوخ کیے جا سکتے ہیں۔

## کوالٹی آف سروس (QoS)

ROS2 کمیونیکیشن کی قابل اعتمادی اور کارکردگی کو ترتیب دینے کے لیے QoS پالیسیاں استعمال کرتا ہے۔

```python
from rclpy.qos import QoSProfile, ReliabilityPolicy, HistoryPolicy, DurabilityPolicy

# سینسر ڈیٹا پروفائل - تازہ ترین ڈیٹا کو ترجیح دیتا ہے
sensor_qos = QoSProfile(
    reliability=ReliabilityPolicy.BEST_EFFORT,
    history=HistoryPolicy.KEEP_LAST,
    depth=1,
    durability=DurabilityPolicy.VOLATILE
)

# قابل اعتماد پروفائل - پیغام کی ترسیل کو یقینی بناتا ہے
reliable_qos = QoSProfile(
    reliability=ReliabilityPolicy.RELIABLE,
    history=HistoryPolicy.KEEP_LAST,
    depth=10,
    durability=DurabilityPolicy.TRANSIENT_LOCAL
)
```

## خلاصہ

- **ROS2** روبوٹ ایپلیکیشنز بنانے کے لیے معیاری مڈل ویئر ہے
- **نوڈز** بنیادی کمپیوٹیشن اکائیاں ہیں
- **ٹاپکس** غیر مطابقت پذیر پبلش-سبسکرائب کمیونیکیشن کو فعال کرتے ہیں
- **سروسز** مطابقت پذیر ریکویسٹ-ریسپانس پیٹرنز فراہم کرتی ہیں
- **ایکشنز** فیڈبیک کے ساتھ طویل عرصے سے چلنے والے، قابل منسوخ کاموں کو سنبھالتے ہیں
- **پیکجز** کوڈ، پیغامات، اور ترتیبات کو منظم کرتے ہیں
- **QoS** پالیسیاں کمیونیکیشن کی قابل اعتمادی کو ترتیب دیتی ہیں
